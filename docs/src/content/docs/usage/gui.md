CLI 引数を手で組み立てる代わりに、ブラウザから設定・起動/停止・CA生成・ライブログ閲覧ができます。

```bash
./jheader-proxy --gui
```

起動すると `http://127.0.0.1:9090` で管理画面が立ち上がり、ブラウザが自動で開きます。

## 画面でできること

- **対象ドメイン・付与ヘッダー・許可クライアント**などをフォームで入力（行を追加して複数指定可）
- **開始 / 停止** ボタンでプロキシを起動・停止（残り時間も表示）
- **CA 証明書の生成**と「ファイルの場所を開く」（macOS は Finder、Windows はエクスプローラー、Linux はファイルマネージャ）
- iPhone に設定する**サーバ（LAN IP）とポートの表示**
- **リクエストログのライブ表示**（SSE）

## 仕組み

管理画面とプロキシ本体は別々のポートで待ち受けます。

```text
ブラウザ ──► 管理画面 127.0.0.1:9090  （localhost 限定）
                  │ 開始/停止
                  ▼
iPhone ─────► プロキシ本体 :8080      （iPhone が Wi-Fi プロキシで接続）
```

- プロキシ本体は最初は停止状態で、画面の「開始」ボタンで起動します。
- **ブラウザを閉じてもプロキシは動き続けます**（画面が消えるだけ）。再度 `http://127.0.0.1:9090` を開けば状態が見えます。
- ターミナルで `Ctrl+C` すると、管理画面とプロキシの両方が穏当に停止します。

## 設定の永続化

起動成功時に、フォームの内容（対象ドメイン・ヘッダー・CAパス・各オプション）が自動保存され、次回 `--gui` 起動時にフォームへ復元されます。

- 保存先: `~/Library/Application Support/jheader-proxy/config.json`
- パーミッション: `0600`（ヘッダー値に認証トークン等が入りうるため）

:::note[保存されるのは「直近の1件」だけ]
名前付きプロファイルの切り替えには対応していません。常に最後に開始した設定が復元されます。
:::

## CA と iPhone 接続

CA が未生成の場合は、CA パスを入力して「CA 生成」ボタンで作成できます（既定パスは設定ディレクトリ配下に補完されます）。

1. 「CA 生成」→ 「ファイルの場所を開く」で証明書のフォルダを開き、AirDrop などで iPhone に送る
2. iPhone でプロファイルをインストールし、**証明書信頼設定を ON** にする（[詳細](/jheader-proxy/usage/ca/)）
3. 画面に表示される**サーバ（LAN IP）とポート**を、iPhone の Wi-Fi プロキシに手動設定する

## オプション

| 引数 | 説明 |
| --- | --- |
| `--gui` | ローカル Web 管理画面を起動する |
| `--gui-listen` | 管理画面の待受アドレス。デフォルトは `127.0.0.1:9090`。任意に変更可 |
| `--no-open` | `--gui` 時にブラウザを自動起動しない |

プロキシ本体のポートは、画面の「待受アドレス」欄で任意に変更できます。

:::caution[ポートが使用中のとき]
`--gui-listen` で指定したポートや、プロキシ本体のポートが使用中の場合、別ポートへ自動でずらさず**エラーになります**。指定を尊重するため、空いているポートを指定し直してください。
:::

## セキュリティ

- 管理画面は **`127.0.0.1` 限定**で待ち受けます。LAN や iPhone からは触れません。
- API は**起動毎に発行されるランダムトークン**で保護されます（画面に埋め込まれ、ブラウザが自動で付与します）。

検証が終わったら、iPhone 側の Wi-Fi プロキシ設定と証明書信頼設定を **OFF** に戻してください（[セキュリティ](/jheader-proxy/guides/security/)）。
