---
title: セキュリティ
description: MITM プロキシを安全に使うための注意点
---

このツールは HTTPS を MITM（復号）するため、扱いを誤ると通信が第三者に漏れ得ます。以下を必ず守ってください。

## 検証が終わったら戻す

- iPhone の Wi-Fi プロキシ設定を **OFF** にする
- 証明書信頼設定（設定 → 一般 → 情報 → 証明書信頼設定）を **OFF** にする
- 不要になった CA 証明書（プロファイル）を削除する

:::danger
CA を信頼している端末は、その秘密鍵を持つ者に全 HTTPS 通信を MITM され得ます。信頼できないネットワーク（公衆 Wi-Fi など）に接続する前に、必ず信頼を OFF にしてください。
:::

## 自分専用CAを使う

組み込みCAは秘密鍵が公開されており危険なため使いません。必ず `--gen-ca` で自分専用 CA を生成し、秘密鍵（`jheader-proxy-ca-key.pem`）は **Git 管理・共有しない**でください。

## クライアントを限定する

`--listen ":8080"` は全インターフェースで待ち受けます。共有 Wi-Fi では `--allow` で iPhone の IP に限定し、第三者にプロキシを踏み台利用される事故を防ぎます。

## 機密ヘッダーの扱い

`Authorization` / `Cookie` / `X-Api-Key` などは機密情報です。起動ログでは自動マスクされますが、扱う場合はログの取り扱いに注意してください。`--redact` で全値をマスクできます。

## 自動停止

停止忘れを防ぐため、既定で起動から **10分** で自動停止します（`--duration`）。長時間使う場合のみ延長してください。

## 想定しない用途

本番環境、不特定多数が使う公開プロキシとしての利用は想定していません。
